# My SQL Project: E-Commerce Funnel Analysis

Hey, this is my first big SQL project! I wanted to analyze an e-commerce dataset to find out where users are dropping off and what's actually working.

I worked with 3 tables – **8,889 events**, **818 orders**, and **200 products**.

## What I used
- PostgreSQL for queries
- DBeaver to run SQL and create charts
- GitHub to share this

## What I found (some surprising stuff)

1. **100% of purchases happen within 1 hour** of the user's first visit.  
   I was honestly surprised. It means this is an impulse-buy audience. If you don't convert them on the first page view, they're probably gone. No point in running "win-back" emails here.

2. **Desktop converts 2x better than Mobile.**  
   Big opportunity here – if the site optimizes the mobile checkout flow, they could easily boost sales.

3. **Organic traffic converts the most**, but **Email brings the highest revenue per user**.  
   So SEO is great for volume, but Email is where the big spenders come from.

4. **User engagement drops after 1 event.**  
   Most people just view one page and leave. Better landing pages could help.

5. **Weekends perform better than weekdays.**  
   Might be worth testing weekend flash sales.
